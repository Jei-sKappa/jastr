# `jastr validate <template-ref>` Implementation Plan

**Goal:** Add a third top-level command, `jastr validate <template-ref>`, that runs the existing static-validation pipeline against a template and reports pass/fail, taking no inputs, requiring no `--out`, and writing nothing.

**Architecture:** Pure reuse inside `@jastr/cli`. A new `executeValidate` in `packages/cli/src/commands.ts` composes the same engine calls (`parseTemplateSource`, `validateTemplateSchema`, `renderTemplateSource`) and CLI helpers (`loadTemplateReference`, `loadProjectConfigVariant`, `sampleInputsForStaticRender`, `validateAgentSkillTarget`/`validateAgentSkillFrontmatter`) that `run` and `generate` already use, so the three commands report the same defect identically. A new `makeValidateCommand` is wired into the Commander program, and `packages/cli/src/args.ts` gains a `validate` entry in its argv allow-list plus a `validateValidateArgs` shape check. `@jastr/engine` gains nothing — no new exports, no new error codes, no new dependencies.

**Tech Stack:** TypeScript, Bun (runtime/bundler/test runner via Vitest), Commander (`@commander-js/extra-typings`), `execa` for CLI integration tests, YAML-driven e2e cases with a living-docs generator.

### What "static validation" already does

The engine + CLI already own the whole validation pipeline. It is reachable today only as a side effect of `run` (validate → render to stdout) and `generate agent-skill` (validate → write a wrapper). `validate` exposes the same pipeline as a standalone command. The pipeline, in order, is:

1. **Load** the template for the ref (`loadTemplateReference`). Direct `.md` refs never read `.jastr/config.yml`.
2. **Parse** frontmatter (`parseTemplateSource`) then **validate the schema** (`validateTemplateSchema`).
3. For a `#<variant-id>` ref, **resolve the selected variant** config (`loadProjectConfigVariant`), which raises `variant_not_found` when absent and `invalid_config` for a malformed/unknown selected shape.
4. **Static render** (`renderTemplateSource`) with *sampled* synthetic inputs (`sampleInputsForStaticRender`) plus real selected locked values. This exercises directive parsing, condition parsing, condition/interpolation reference checks, include resolution + containment, missing-include detection, include-cycle detection, and engine input validation. It is the same call `generate` makes; it does **not** assert which branch a real invocation would take.
5. **Agent-skill target metadata** validation — but only when the resolved ref declares it (the template's `targets.agent-skill`, overlaid by any selected variant `agent-skill.frontmatter`). When nothing declares agent-skill metadata, this step is skipped and is **not** an error.

> **Step order matters for `validate`.** The spec orders the static render (step 4) *before* agent-skill target validation (step 5). This is the reverse of `executeGenerate`, which validates the target first because it is about to write one. For `validate`, follow the spec order: render, then target metadata. (For a single-defect template the order is invisible; we follow the spec to be exact.)

```ts
function validateValidateArgs(rest: string[]): void {
  let sawRef = false;
  for (const arg of rest) {
    if (isHelpToken(arg)) return;
    if (arg.startsWith("--")) {
      throw new JastrError(
        "invalid_command",
        `Unknown validate option ${arg}.`,
      );
    }
    if (sawRef) {
      throw new JastrError(
        "invalid_command",
        `Invalid validate argument ${arg}.`,
      );
    }
    sawRef = true;
  }
  if (!sawRef) {
    throw new JastrError(
      "invalid_command",
      "Missing template reference for validate.",
    );
  }
}
```

```ts
export async function executeValidate(opts: {
  templateRef: string;
  cwd: string;
}): Promise<string> {
  const template = await loadTemplateReference({
    cwd: opts.cwd,
    templateRef: opts.templateRef,
  });
  const parsed = parseTemplateSource(template.source);
  const schema = validateTemplateSchema(parsed.frontmatter);

  const selectedVariant =
    template.mode === "named" && template.variantId !== undefined
      ? await loadProjectConfigVariant({
          projectRoot: template.projectRoot,
          templateRef: template.templateRef,
          variantId: template.variantId,
        })
      : undefined;

  // Static render exercises directives, conditions, interpolation references,
  // include resolution/containment, missing-include and cycle detection, and
  // engine input validation over sampled values (plus selected locked values).
  await renderTemplateSource({
    source: template.source,
    sourceId: path.relative(template.cwd, template.templatePath),
    inputs: sampleInputsForStaticRender(schema, selectedVariant?.lockedInputs),
    includeResolver: createFileIncludeResolver(template),
  });

  // Agent-skill target metadata is validated only when the resolved ref
  // declares it. validate never *requires* a target, so a ref with no
  // agent-skill metadata anywhere is still valid.
  validateDeclaredAgentSkillTarget(schema, selectedVariant);

  return `Template ${opts.templateRef} is valid.`;
}

function validateDeclaredAgentSkillTarget(
  schema: TemplateSchema,
  selectedVariant: ProjectConfigVariant | undefined,
): void {
  const declaredTarget = schema.targets["agent-skill"];

  if (selectedVariant === undefined) {
    if (declaredTarget === undefined) return;
    validateAgentSkillTarget(declaredTarget);
    return;
  }

  if (
    declaredTarget === undefined &&
    selectedVariant.agentSkillFrontmatter === undefined
  ) {
    return;
  }

  validateAgentSkillFrontmatter({
    ...readOptionalAgentSkillFrontmatter(declaredTarget),
    ...(selectedVariant.agentSkillFrontmatter ?? {}),
  });
}
```

```ts
import { Command } from "@commander-js/extra-typings";
import { executeValidate } from "../commands";

export function makeValidateCommand() {
  return new Command("validate")
    .description("Validate a Jastr template without rendering or writing output")
    .argument(
      "<template-ref>",
      "Template id, template variant (<id>#<variant>), or .md file path",
    )
    .configureOutput({ outputError: () => {} })
    .exitOverride()
    .action(async (templateRef) => {
      const output = await executeValidate({
        templateRef,
        cwd: process.cwd(),
      });
      process.stdout.write(output);
    });
}
```
