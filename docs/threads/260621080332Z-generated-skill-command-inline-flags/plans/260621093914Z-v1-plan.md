# Generated Agent Skill — Inline Input Flags Implementation Plan

**Goal:** Make `jastr generate agent-skill` inline a template's *required* inputs into the generated `jastr run …` command as `--<name>=<value>` placeholders, and render single-input wrappers as a tailored sentence with no `## Inputs` section.

**Architecture:** The change is entirely inside `@jastr/cli`'s generator (`buildAgentSkillContent` in `packages/cli/src/targets/agent-skill.ts`). The function already receives the *rendered* input list; it now selects one of four body shapes (A: no inputs, B: one required input, C: one optional input, D: two or more inputs) by that list's length, builds the command by appending `--<name>=<value>` for each required input in declaration order, and reuses a shared per-input descriptor routine for both the Shape D bullet and the single-input sentence. The engine, schema, error codes, frontmatter pass-through, and call site are untouched.

**Tech Stack:** TypeScript (Node-compatible, no Bun runtime APIs), Bun (runtime/bundler/test runner via Vitest), Biome (format/lint), `yaml` (frontmatter serialization). E2e cases are YAML manifests run against the real CLI; functional requirements are YAML; `BEHAVIOR.md` is generated from both.

## Key facts you must not get wrong

- **Byte-exactness is the contract.** `--check` compares raw bytes with no normalization (`packages/cli/src/targets/agent-skill.ts:304`). Every generated file ends with exactly one trailing newline (`…stop.\n`). Every space in the pinned sentences is a regular space U+0020; the separator before a description is ` — ` (space, em dash **U+2014**, space).
- **The placeholder is the literal token `<value>` for every type**, including `boolean`. Never emit a bare `--flag` form.
- **Only `required` inputs are inlined** into the command, in template declaration order. Optional inputs never appear in the command.
- **`description` is verbatim, no escaping.** Enum values are joined with `|` (pipe), in declared order.
- **`noUncheckedIndexedAccess` is on** (`tsconfig.base.json:15`). Indexing `inputs[0]` yields `T | undefined`; the implementation below avoids that with array destructuring + a truthiness guard.
- The engine is **not** touched. No new `JastrErrorCode`. No frontmatter change. No call-site change.

## Why some cases change shape (read before Task 4)

After the implementation, the generator output for a template's *rendered* input list changes as follows. The "rendered" list is what `listUnlockedTemplateInputs` returns: **all** declared inputs for a non-variant ref, the **unlocked** inputs for a variant ref.

- `generate-passthrough` (`demo`, one required enum `language`) → **Shape B** (no `## Inputs`; tailored sentence; command `jastr run demo --language=<value>`).
- `generate-variant-unlocked` (`review#deep`, unlocked = one required string `language`) → **Shape B** (`jastr run review#deep --language=<value>`).
- `generate-variant-inputs-hidden` (`review#deep`, today unlocked = one required string `mode`) → would collapse to Shape B and **lose its `## Inputs` section**, which is the only place `VAR-FR-0009.AC-0001` ("lists an unlocked input … in the Inputs section") is demonstrated. We therefore **add a second unlocked input (`language`)** to its template so it stays **Shape D** — keeping the case's purpose (show the section while hiding the locked input) meaningful and keeping that AC covered.
- `generate-inputs-section` (`deploy`, 5 inputs incl. required `env` + `tag`) → stays **Shape D**; only the command changes to `jastr run deploy --env=<value> --tag=<value>`.
- `generate-check-variant` (`review#deep`, unlocked = one required string `language`) → its committed `fixture/out/SKILL.md` must become Shape B so `--check` reports up-to-date.

Two existing acceptance criteria go stale because exactly-one-unlocked no longer renders a section:

- `GEN-FR-0011.AC-0002` ("one or more inputs → Inputs section + bare command") → reworded to "two or more inputs → Inputs section", moved to `generate-inputs-section`.
- `GEN-FR-0012.AC-0004` ("Inputs section when at least one declared input is not locked") → reworded to "two or more declared inputs not locked", moved to `generate-variant-inputs-hidden`.

```ts
function inputDescriptor(definition: TemplateInputDefinition): {
  typeToken: string;
  defaultSeg: string;
  descSeg: string;
} {
  const typeToken =
    definition.type === "enum"
      ? `enum: ${definition.values.join("|")}`
      : definition.type;
  const defaultSeg =
    !definition.required && definition.default !== undefined
      ? `, default: ${String(definition.default)}`
      : "";
  const descSeg =
    definition.description !== undefined ? ` — ${definition.description}` : "";
  return { typeToken, defaultSeg, descSeg };
}

function renderInputBullet(
  name: string,
  definition: TemplateInputDefinition,
): string {
  const { typeToken, defaultSeg, descSeg } = inputDescriptor(definition);
  const reqToken = definition.required ? "required" : "optional";
  return `- \`--${name}\` (${typeToken}, ${reqToken}${defaultSeg})${descSeg}`;
}
```

```ts
function buildCommand(
  templateRef: string,
  inputs: ReadonlyArray<{ name: string; definition: TemplateInputDefinition }>,
): string {
  let command = `jastr run ${templateRef}`;
  for (const { name, definition } of inputs) {
    if (definition.required) command += ` --${name}=<value>`;
  }
  return command;
}

function renderSingleInputSentence(input: {
  name: string;
  definition: TemplateInputDefinition;
}): string {
  const { name, definition } = input;
  const { typeToken, defaultSeg, descSeg } = inputDescriptor(definition);
  if (definition.required) {
    return `This skill takes one input, \`--${name}\` (${typeToken})${descSeg}. Fill in \`--${name}=<value>\` from the user's request. Then run this command and follow its output exactly:`;
  }
  return `This skill takes one optional input, \`--${name}\` (${typeToken}${defaultSeg})${descSeg}. Add \`--${name}=<value>\` if the user's request calls for it; otherwise leave it out. Then run this command and follow its output exactly:`;
}
```

- [ ] **Step 4: Rewrite `buildAgentSkillContent` to select the four shapes**

Replace the entire existing `buildAgentSkillContent` function (lines 182-230) with:

```ts
export function buildAgentSkillContent(options: {
  templateRef: string;
  target: AgentSkillTarget;
  inputs: ReadonlyArray<{ name: string; definition: TemplateInputDefinition }>;
}): string {
  const frontmatter = {
    name: options.target.name,
    description: options.target.description,
    ...options.target.frontmatter,
  };
  const header = `---\n${YAML.stringify(frontmatter).trimEnd()}\n---`;
  const command = buildCommand(options.templateRef, options.inputs);

  // Shape A — no rendered inputs.
  if (options.inputs.length === 0) {
    return `${header}

Run this command and follow its output exactly:

\`\`\`bash
${command}
\`\`\`

${AGENT_SKILL_FAILURE_LINE}
`;
  }

  // Shapes B/C — exactly one rendered input. Destructuring + the truthiness
  // guard narrows `single` to a defined value under noUncheckedIndexedAccess.
  const [single, ...rest] = options.inputs;
  if (single !== undefined && rest.length === 0) {
    return `${header}

${renderSingleInputSentence(single)}

\`\`\`bash
${command}
\`\`\`

${AGENT_SKILL_FAILURE_LINE}
`;
  }

  // Shape D — two or more rendered inputs.
  const bullets = options.inputs
    .map(({ name, definition }) => renderInputBullet(name, definition))
    .join("\n");

  return `${header}

## Inputs

${bullets}

Map the user's request to the inputs above and append them as \`--flag=value\` arguments, including every required input. Then run this command and follow its output exactly:

\`\`\`bash
${command}
\`\`\`

${AGENT_SKILL_FAILURE_LINE}
`;
}
```

Note: the `header` string is `---\n<frontmatter>\n---`, so `${header}` followed by a blank line reproduces the original `---\n<frontmatter>\n---\n\n…` bytes exactly.
