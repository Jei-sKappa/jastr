# Seed: jastr install — fetch and install remote templates into `.jastr/`
External: none

Backlog idea, not yet settled: a `jastr install` (name TBD) command that fetches a template from a remote source and installs it into the local or global `.jastr/` root. Heavily inspired by `.library/sources/vercel-labs_skills` — the vercel-labs `skills` CLI, where `npx skills add owner/repo` resolves many source formats (GitHub shorthand, full GitHub/GitLab/git URLs, direct path to a skill in a repo, local path) and installs to a project or `--global` user directory, with `--copy` vs symlink and per-skill selection. The jastr analogue would resolve a remote template source, then land it under `.jastr/<template-id>/TEMPLATE.md` (or a grouped/global layout), reusing the existing dual-root local/global discovery. Open questions to settle downstream: source-format set, fetch mechanism, local-vs-global target, naming/group placement, update/refresh semantics, conflict-with-existing handling, and trust of remote content.
