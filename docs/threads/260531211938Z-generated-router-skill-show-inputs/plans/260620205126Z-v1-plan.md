# Generated Agent Skill — show available template inputs Implementation Plan

**Goal:** Make `jastr generate agent-skill` emit a `## Inputs` section listing each
available flag (name, type, required/optional, default, optional description) in the
generated wrapper body, add an optional per-input `description` key to the template
schema, and remove the `$ARGUMENTS` token entirely.

**Architecture:** The engine (`@jastr/engine`) gains a pure schema-validation rule for
the new optional `description` key; nothing else in the engine changes. The CLI
(`@jastr/cli`) owns all body rendering: `buildAgentSkillContent` is reshaped to take an
ordered list of inputs and emit one of two byte-stable body shapes (lean vs. `## Inputs`),
and `executeGenerate` computes that ordered list (all inputs for a plain ref, only the
unlocked inputs for a variant ref). Functional requirements, e2e cases, and docs are
reconciled so the stale `$ARGUMENTS` contract is replaced.

**Tech Stack:** TypeScript, Bun (runtime/bundler), Vitest, Biome, `yaml`, a YAML-driven
e2e case harness with requirement traceability.

## Background you need before starting

You have probably never seen this codebase. Read this section once; it saves you from
guessing.

- **Two packages in one workspace.** `packages/engine` is pure (no Node FS, no CLI, no
  `$ARGUMENTS` knowledge). `packages/cli` owns the `jastr` binary, file I/O, and all Agent
  Skill body rendering. Do **not** add body-rendering or `$ARGUMENTS` logic to the engine.
- **The generated wrapper today.** For an input-bearing template the CLI writes a
  `SKILL.md` whose command is `jastr run <ref> $ARGUMENTS`; for a no-input template it
  writes `jastr run <ref>`. `$ARGUMENTS` is the slash-command substitution token and is
  actively wrong for natural-language invocation. This spec deletes it. The command is
  always the bare `jastr run <ref>`; an input-bearing template instead gets a `## Inputs`
  section plus a one-sentence instruction telling the agent to construct `--flag=value`
  arguments.
- **`--check` does an exact-byte comparison.** Generation must be deterministic. Every
  separator, the em dash (U+2014, bytes `e2 80 94`), the bullet order (template
  declaration order), and the instruction sentence are pinned. A single byte drift makes
  `--check` report "stale" and fails CI.
- **The e2e harness enforces traceability.** `packages/cli/test/e2e/harness/traceability.ts`
  throws if any **active** acceptance criterion (AC) is not covered by at least one e2e
  case's `covers:` list. So: every new AC you add to a `requirements/functional/*.yml`
  file **must** be covered by a case in the same change, or `bun run test` fails. Likewise
  a case may not `covers:` a ref that does not exist.
- **An e2e case's `fixture/` contents become the project root.** A case directory holds
  `case.yml`, an optional `fixture/` (copied to a temp dir that becomes the CWD/project
  root), and an optional `expected/` (files compared byte-for-byte). There is no `fixture/`
  directory at runtime — its *contents* land at the root, so `fixture/.jastr/...` becomes
  `<root>/.jastr/...`.
- **Why some commits update fixtures, not just expected files.** A `--check` case keeps the
  committed wrapper it checks *inside* `fixture/out/SKILL.md` (the file the CLI reads), not
  in `expected/`. Changing generator output means updating that fixture file too, or
  `--check` reports stale.

In `packages/engine/src/schema.ts`, add this function immediately after `validateEnumDefault`

```ts
function validateOptionalDescription(
  inputName: string,
  rawDefinition: Record<string, unknown>,
): string | undefined {
  if (!Object.hasOwn(rawDefinition, "description")) return undefined;
  const value = rawDefinition.description;
  if (typeof value !== "string") {
    throw new JastrError(
      "malformed_schema",
      `Description for input ${inputName} must be a string.`,
      { inputName },
    );
  }
  if (value.trim() === "") {
    throw new JastrError(
      "malformed_schema",
      `Description for input ${inputName} cannot be empty.`,
      { inputName },
    );
  }
  if (value.includes("\n") || value.includes("\r")) {
    throw new JastrError(
      "malformed_schema",
      `Description for input ${inputName} must be a single line.`,
      { inputName },
    );
  }
  return value;
}
```

Note: the original (untrimmed) string is returned so the generator echoes it verbatim.
No maximum length is imposed.

```ts
export function buildAgentSkillContent(options: {
  templateRef: string;
  target: AgentSkillTarget;
  inputs: ReadonlyArray<{ name: string; definition: TemplateInputDefinition }>;
}): string {
  const frontmatter = {
    name: options.target.name,
    description: options.target.description,
    ...options.target.frontmatter,
  };
  const frontmatterSource = YAML.stringify(frontmatter).trimEnd();
  const command = `jastr run ${options.templateRef}`;

  if (options.inputs.length === 0) {
    return `---
${frontmatterSource}
---

Run this command and follow its output exactly:

\`\`\`bash
${command}
\`\`\`

If the command exits non-zero, report the exact error output to the user and stop.
`;
  }

  const bullets = options.inputs
    .map(({ name, definition }) => renderInputBullet(name, definition))
    .join("\n");

  return `---
${frontmatterSource}
---

## Inputs

${bullets}

Map the user's request to the inputs above and append them as \`--flag=value\` arguments, including every required input. Then run this command and follow its output exactly:

\`\`\`bash
${command}
\`\`\`

If the command exits non-zero, report the exact error output to the user and stop.
`;
}

function renderInputBullet(
  name: string,
  definition: TemplateInputDefinition,
): string {
  const typeToken =
    definition.type === "enum"
      ? `enum: ${definition.values.join("|")}`
      : definition.type;
  const reqToken = definition.required ? "required" : "optional";
  const defaultSeg =
    !definition.required && definition.default !== undefined
      ? `, default: ${String(definition.default)}`
      : "";
  const descSeg =
    definition.description !== undefined ? ` — ${definition.description}` : "";
  return `- \`--${name}\` (${typeToken}, ${reqToken}${defaultSeg})${descSeg}`;
```
