# Seed: Living-docs generator ignores `global-fixture/`
External: none — private single-owner repo (no team tracker); follow-up gap from the home-global `.jastr` thread `docs/threads/260621200326Z-home-global-jastr-dir/`.

The home-directory global `.jastr` work added a `global-fixture/` folder to e2e cases (the per-case `JASTR_HOME` global root) and taught the e2e *runner* about it, but the living-docs generator (`packages/cli/scripts/living-docs.ts`) was never taught about it: `loadRenderCases` reads only each case's `fixture/`, and `renderInputSection` prints "_Empty — no .jastr/ directory present_" whenever the local fixture is empty. So in `packages/cli/docs/BEHAVIOR.md`, all 17 new cases that use a `global-fixture/` render without their global root — 5 (e.g. `global-only-resolve`) show a misleading "Empty" input then produce global output from nowhere, and 12 (e.g. `local-shadows-global`) silently drop the global half. `docs:cli:living --check` still passes because it only verifies the (blind) generator is deterministic, not that its output is complete.

Proposed fix: add a `globalInputFiles` field to `RenderCase`; have `loadRenderCases` read `global-fixture/` with the same recursive walk as `fixture/`; in `renderInputSection` render two labelled trees (Local project + Global root `$JASTR_HOME/.jastr`), printing "Empty" only when both roots are empty; add `living-docs.test.ts` coverage; regenerate `BEHAVIOR.md` and confirm `--check` passes.
